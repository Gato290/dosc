BackgroundVideo

Support dua tipe input: Asset ID dipakai langsung tanpa download, sedangkan URL di-download dan di-cache otomatis menggunakan getcustomasset. Tersedia 4 method untuk mengontrolnya setelah window dibuat:

-- Cara 1: Asset ID langsung local Window = VelarisUI:Window({ BackgroundVideo = "rbxassetid://123456789", BackgroundVideoOverlay = 0.4, }) -- Cara 2: URL (download + cache otomatis) local Window = VelarisUI:Window({ BackgroundVideo = "https://example.com/bg.webm", BackgroundVideoOverlay = 0.4, -- 0 = solid, 1 = transparan }) Window:SetBackgroundVideoVolume(0) -- mute Window:PauseBackgroundVideo() Window:PlayBackgroundVideo() Window:SetBackgroundVideoOverlay(0.6) -- lebih gelap
⚠️ Method BackgroundVideo hanya tersedia setelah video selesai di-load. Jika pakai URL, video di-download sekali dan di-cache di file lokal VelarisUI_bgvideo.webm. Jika pakai Asset ID, video langsung diputar tanpa proses download.

Tutorial: Cara mendapatkan URL webm

Jika kamu ingin pakai file video sebagai background (cara URL), ikuti langkah berikut:

#LangkahLink
1 Buka Convertio dan upload file video kamu (mp4, gif, dll). Pilih format output webm, lalu klik Convert dan download hasilnya. convertio.co/id
2 Buka Catbox — file host gratis. Upload file .webm yang sudah di-download tadi. catbox.moe
3 Setelah upload selesai, copy URL yang diberikan Catbox (contoh: https://files.catbox.moe/xxxxxx.webm). Pakai URL itu di BackgroundVideo. —
local Window = VelarisUI:Window({ BackgroundVideo = "https://files.catbox.moe/xxxxxx.webm", BackgroundVideoOverlay = 0.4, })

Key System

Tambahkan key system ke dalam konfigurasi Window. UI akan menunggu verifikasi key sebelum window utama muncul. Key system sekarang menggunakan format baru dengan Links, Steps, dan Callback langsung — tidak lagi menggunakan Buttons.

KeySystem = { Title = "My Hub", Icon = "lucide:bird", Placeholder = "Enter your key here", Default = "", DiscordText = "Join the Discord", DiscordUrl = "https://discord.gg/xxxxx", Links = { { Name = "Linkvertise", Icon = "lucide:link", Url = "https://linkvertise.com/xxxxx" }, { Name = "LootLabs", Icon = "lucide:gift", Url = "https://lootlabs.gg/xxxxx" }, { Name = "Work.ink", Icon = "lucide:pen", Url = "https://work.ink/xxxxx" }, }, Steps = { "Choose your Authorization System!", "Complete ads ( 2 Checkpoint )", "Copy and paste key here!!", }, Callback = function(key) local validKeys = { "mykey123", "vip-key-456", } for _, v in ipairs(validKeys) do if key == v then return true -- ✓ key valid → buka window end end return false -- ✗ key salah → shake + highlight input merah end, },

Tag API

Window:Tag() mengembalikan sebuah objek API yang bisa digunakan untuk mengubah tag secara dinamis setelah dibuat.

local MyTag = Window:Tag({ Title = "v0.0.1", Icon = "lucide:tag", Color = "Amber", }) -- Update teks tag MyTag:SetTitle("v0.0.2") -- Ganti icon MyTag:SetIcon("lucide:star") MyTag:SetIcon("") -- sembunyikan icon -- Ganti warna (nama tema / hex / Color3) MyTag:SetColor("Rose") MyTag:SetColor("#FF5500") MyTag:SetColor(Color3.fromRGB(48, 255, 106)) -- Hapus tag dari topbar MyTag:Destroy()
MethodDescription
:SetTitle(text)Ubah teks label tag
:SetIcon(iconName)Ubah icon tag. Kirim "" atau nil untuk menyembunyikan icon
:SetColor(colorInput)Ubah warna tag (nama tema, hex "#RRGGBB", atau Color3). Animasi tween otomatis
:Destroy()Hapus tag dari topbar sepenuhnya

Gunakan :AddTabbox() untuk membuat group tab bertingkat di dalam sebuah section biasa. Setiap kali :AddTabbox() dipanggil, satu tab baru ditambahkan ke group yang sama. Method ini mengembalikan Items API — tambahkan elemen ke dalamnya seperti section biasa.

💡 Semua pemanggilan :AddTabbox() di dalam Section yang sama akan otomatis digabungkan ke satu group tabbox. Tab pertama aktif secara default. Jika jumlah tab melebihi lebar section, tab bar bisa di-scroll horizontal.

Contoh Dasar

local Section = Tabs.Main:AddSection({ Title = "Combat" }) local Tab1 = Section:AddTabbox({ Title = "Auto Farm" }) local Tab2 = Section:AddTabbox({ Title = "Settings" }) local Tab3 = Section:AddTabbox({ Title = "Info", Icon = "lucide:info" }) Tab1:AddToggle({ Title = "Auto Collect", Default = false, Callback = function(v) end }) Tab2:AddSlider({ Title = "Speed", Min = 1, Max = 100, Default = 16, Callback = function(v) end }) Tab3:AddParagraph({ Title = "About", Content = "VelarisUI v0.1.9" })

ConfigSection
Sistem manajemen config siap pakai — save, load, delete, autoload

VelarisUI:AddConfigSection(Tab, SectionConfig) menambahkan section config lengkap ke Tab yang dipilih. Semua UI sudah disediakan otomatis: input nama, dropdown list, tombol Create/Load/Overwrite/Delete/Refresh, Set Autoload, dan label autoload aktif.

-- Cara 1: tanpa argumen kedua (default name "Configuration") VelarisUI:AddConfigSection(Tabs.Config) -- Cara 2: string langsung sebagai nama section VelarisUI:AddConfigSection(Tabs.Config, "My Config") -- Cara 3: table dengan nama + icon VelarisUI:AddConfigSection(Tabs.Config, { Name = "My Config", Icon = "lucide:settings-2", })

Parameters

ParameterTypeDescription
TabTab objectTab yang dikembalikan dari Window:AddTab()
SectionConfigstring | table | nilNama section (string), atau table dengan Name dan Icon, atau biarkan kosong untuk default

Fitur yang disediakan otomatis

ElemenFungsi
Input "Config name"Ketik nama config baru sebelum Create
Button "Create config"Buat file config baru dengan nama dari input. Nama di-sanitize otomatis
Dropdown "Config list"Daftar semua file config yang tersimpan
Button "Load config"Load config yang dipilih di dropdown ke semua elemen
Button "Overwrite config"Simpan state elemen saat ini ke config yang dipilih (overwrite)
Button "Delete config"Hapus config yang dipilih. Jika config adalah autoload, autoload ikut di-reset
Button "Refresh list"Refresh dropdown list dari folder config
Button "Set as autoload"Set config terpilih sebagai yang otomatis di-load saat script dijalankan
Button "Reset autoload"Hapus autoload
Paragraph "Current autoload config"Menampilkan nama config autoload aktif, atau "none"

Perilaku Autoload

Jika ada config yang di-set sebagai autoload, config tersebut akan otomatis di-load saat AddConfigSection dipanggil (dengan delay 0.5 detik). File autoload disimpan di Configname/autoload.txt.

💡 Config disimpan di folder Configname/Config/NamaConfig.json. Jika Version config tidak cocok dengan versi script saat ini, load akan dibatalkan dan user diberitahu via notifikasi.

Contoh lengkap

local Window = VelarisUI:Window({ Title = "My Hub", Configname = "MyHub", Version = 1, }) local Tabs = { Main = Window:AddTab({ Name = "Main", Icon = "lucide:home" }), Config = Window:AddTab({ Name = "Config", Icon = "lucide:folder" }), } -- Tambah elemen di tab Main... -- Config section di tab Config VelarisUI:AddConfigSection(Tabs.Config, { Name = "Configuration", Icon = "lucide:settings-2", })
